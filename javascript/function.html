Example 1:

What's the difference between function declaration and function expression in terms of hoisting?

Function declarations are hoisted and can be called before their definition in the code. Function expressions are not hoisted and must be defined before being called.

Example 14:

What is higer order function?

Ans and Why

A higher-order function is a function that either takes one or more functions as arguments or returns a function as its result.

Higher-order functions allow for more abstract and flexible code by enabling functions to operate on other functions.

Example 18:

What is Closure? When is it created?

Ans and Example

A closure is a function that retains access to its lexical scope even when the function is executed outside that scope. It is created when a function is defined inside another function and the inner function references variables from the outer function.

function outer(){
    let count = 0;
    return function(){
        count++;
        console.log(count);
    };
}

In this example, the inner function returned by outer forms a closure that retains access to the count variable even after outer has finished executing.

Example 20:

Convert below function into an IIFE:

function init(){
    console.log("Initialized");
}

Ans and Why

(function init(){ console.log("Initialized"); })();

An Immediately Invoked Function Expression (IIFE) is a function that is defined and executed immediately. By wrapping the function in parentheses and adding () at the end, we create an IIFE that runs as soon as it is defined.

Example 21:

What is the use of IIFE? Name one real-world use case.

let fun=(function(){
    let score = 0; // this is private variable
    return{
        getScore: function(){
            console.log(score);
        },
        setScore: function(val){
            score = val;
        }
    }
})()

Ans and Why

IIFE is used to create a private scope for variables and functions, preventing them from polluting the global namespace. A real-world use case is creating modules that encapsulate functionality and state, like in the example above where score is a private variable.

The IIFE returns an object with methods to get and set the score, while keeping the score variable private.

Ans and Why

TypeError: temp_var is not a function

In this case, temp_var is declared using var, which hoists the variable declaration but not the assignment. Therefore, when temp_var() is called, it is undefined at that point, leading to a TypeError.
